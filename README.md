# Sales-Performance-Data-Integrity-Analysis
End TO End Data Analytics Project (Python| SQL| Power BI)

##  Project Overview
This project focuses on analyzing a retail sales dataset containing 10,000 transactions. The goal was to transform "dirty" raw data into actionable business insights by identifying revenue trends, payment behavior, and data quality issues.

##  Tech Stack
- **Data Cleaning:** Python (Pandas, NumPy)
- **Data Analysis:** SQL (PostgreSQL)
- **Visualization:** Power BI

## 🔍 Key Insights & Analysis
- **Revenue Trends:** Peak sales observed in June (~7.9K), indicating strong mid-year performance.
- **February Deep Dive:** Despite having the lowest transaction volume (759 orders),But February achieved the 2nd highest **Average Order Value ($9.22)**.
- **Data Integrity:** Identified that **31% of transactions** had missing payment methods, highlighting a critical need for better data tracking at the source.
- **Payment Preferences:** Digital Wallets and Credit Cards are almost equal in usage (~23% each).

##  Data Cleaning Process
Implemented robust cleaning techniques using Python:
- Handled missing values using median imputation for numerical columns.
- Resolved "ERROR" and "UNKNOWN" strings in categorical data.
- Standardized Transaction Date formats using forward and backward filling for consistency.
